1.  Что такое абстрактный класс и чем он отличается от обычного класса?
- Это класс который содержит хотя бы один абстрактный метод. 
Абстрактные методы не имеют реализации в самом классе и должны быть реализованы 
в производных классах.
Обычные классы могут создавать экземпляры (объекты) напрямую, тогда как 
абстрактные классы не могут быть инстанциированы напрямую.

2.  Как получить доступ к абстрактному классу и зачем нужно наследование?
- Доступ к абстрактному классу осуществляется через его производные классы.
Наследование позволяет использовать функциональность абстрактного класса в производных 
классах, при этом обеспечивая реализацию абстрактных методов в этих классах.

3.  Приведите пример наследования от абстрактного класса и создания производного класса. 
Укажите, как переопределить абстрактный метод в производном классе.

abstract class Shape
{
    public abstract void Draw();
}

class Circle : Shape
{
    public override void Draw()
    {
        Console.WriteLine("Drawing a circle");
    }
}

4.  Как ключевое слово "virtual" позволяет переопределять методы в производных классах?
-Ключевое слово virtual позволяет методу быть переопределенным в производных классах.

5.  Можно ли использовать модификатор "virtual" с методами, которые являются статическими, 
абстрактными, приватными или переопределенными? Почему или почему нет?
- Статические методы: Нельзя использовать "virtual" с методами, так как 
статические методы не могут быть переопределены.
Абстрактные методы: Нельзя использовать "virtual", так как абстрактные
 методы уже предназначены для переопределения.
Приватные методы: "virtual" также не может быть использовано, так как 
приватные методы не видны в производных классах.
Переопределенные методы: "virtual" можно использовать, но это не обязательно, 
поскольку методы по умолчанию считаются "virtual".
